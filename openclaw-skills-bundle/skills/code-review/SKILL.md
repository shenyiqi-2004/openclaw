# 代码审查

## 审查清单

### 1. 正确性
- 功能是否符合需求
- 边界条件处理
- 错误处理

### 2. 可读性
- 命名清晰
- 函数单一职责
- 注释必要且简洁

### 3. 性能
- 时间/空间复杂度
- 资源泄漏
- 数据库查询优化

### 4. 安全
- 输入验证
- 敏感数据处理
- SQL注入/XSS防护

### 5. 测试
- 单元测试覆盖
- 测试用例合理

## 审查话术

```markdown
## Code Review

### ✅ 优点
-

### ⚠️ 建议
-

### 🔧 修改建议
```

## 常用工具

- ESLint
- Prettier
- SonarQube
- GitHub PR reviews
