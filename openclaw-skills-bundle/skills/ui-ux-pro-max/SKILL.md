# UI/UX设计指南

## 用户体验原则

1. **可见性**：系统状态可见
2. **匹配**：匹配真实世界
3. **控制**：用户控制与自由
4. **一致**：一致性与标准化
5. **防错**：预防错误
6. **识别**：识别而非记忆
7. **灵活**：灵活与效率
8. **帮助**：帮助与文档

## 无障碍 (WCAG)

### 1. 可感知
- 文字与背景对比度 ≥ 4.5:1
- 图片有alt文本
- 视频有字幕

### 2. 可操作
- 键盘可导航
- 有焦点指示
- 有足够点击区域 (44x44px)

### 3. 可理解
- 语言声明
- 错误提示明确
- 输入标签清晰

### 4. 健壮
- 语义化HTML
- ARIA正确使用
- 兼容性良好

## 交互设计

### 反馈
- 操作反馈：加载/成功/失败
- 等待时间：<100ms理想，<1s可接受
- 进度指示：加载条/骨架屏

### 动效
- 时长：150-300ms
- 曲线：ease-in-out
- 意义：引导注意/状态变化

## 表单设计

- 标签清晰
- 占位符作提示
- 错误信息具体
- 必填项标识 *
- 支持键盘操作
