# Strategy Matrix

## 四种压缩策略对比

| 维度 | summarizer | micro | auto | sessionMemory |
|------|-------------|-------|------|---------------|
| **触发条件** | 手动/转换点/调试完/死路 | idle>30s 或 cache_edits | Token 阈值 circuit breaker | 会话关闭 |
| **压缩效果** | 完整结构化摘要 | 纯裁剪，无摘要 | 自动选择 | 蒸馏到外部记忆 |
| **性能开销** | 高 | 低 | 中 | 中 |
| **head 保护** | ✅ | ✅ | ✅ | ✅ |
| **tail 保护** | ✅ | ✅ | ✅ | ✅ |
| **LLM 调用** | ✅ 必需 | ❌ | 可选 | 可选 |
| **适用场景** | 长程收尾/清场 | 短暂 idle | 被动防溢出 | 会话结束 |

## 保护区

| 区域 | 内容 | 压缩 |
|------|------|------|
| head | system prompt + 首轮交互 | ❌ 禁止 |
| tail | 最近 ~20k token | ❌ 禁止 |
| middle | 中间轮次 | ✅ 可压缩 |

## 触发时机矩阵

| 时机 | 推荐策略 | 强制程度 |
|------|---------|---------|
| 探索→执行转换 | summarizer | 主动 |
| 调试完成后 | summarizer | 主动 |
| 死路放弃后 | summarizer | 主动 |
| idle > 30s | micro | 被动 |
| 会话关闭 | sessionMemory | 被动 |
| Token 阈值达到 | auto | 被动 |
| 实现中途 | ❌ 禁止 | — |
