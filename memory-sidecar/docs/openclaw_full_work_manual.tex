\chapter{System Summary}
The current design is intentionally split into two layers:
\begin{itemize}
  \item \textbf{memory-sidecar}: controller, observability, recovery, runtime state snapshots.
  \item \textbf{memory-lancedb-pro}: canonical long-term memory engine.
\end{itemize}
The old state where sidecar also behaved like a competing retrieval engine is no longer the intended architecture.
\chapter{Architecture Boundary}
\section{What Sidecar Owns}
\begin{itemize}
  \item event durability
  \item commit, ack, replay
  \item mode switching
  \item health, pressure, noise, complexity tracking
  \item reflection and self-check triggering
  \item runtime traceability
  \item operational snapshots such as working, summary, strategy, reflection, selfcheck, and runtime
\end{itemize}
\section{What memory-lancedb-pro Owns}
\begin{itemize}
  \item long-term recall
  \item long-term store, update, forget
  \item hybrid retrieval
  \item rerank
  \item scope isolation
  \item lifecycle and decay on the long-term memory side
\end{itemize}
\section{Explicit Non-Goals For Sidecar}
Sidecar should not rebuild a second long-term memory engine when \texttt{memory-lancedb-pro} is available.

\chapter{Transactional Flow}
\begin{enumerate}
  \item A runtime trigger persists an event before doing any sidecar work.
  \item Request-side runtime stops after durable emit; it does not run a full memory cycle inline.
  \item A worker later claims one queued or replayable event.
  \item Sidecar checks whether that event was already acknowledged or already applied.
  \item If already applied, sidecar performs an idempotent duplicate no-op and writes the proper acknowledgement.
  \item If not yet applied, sidecar runs one signal-driven control cycle and then writes acknowledgement and commit records.
\end{enumerate}
\chapter{Replay And Recovery}
Failed and replayable events remain durable. The next worker pass may replay them. If effects already happened but ack was missing, sidecar detects that by event id and runtime state, then finalizes with a duplicate no-op acknowledgement instead of repeating memory side effects.
\chapter{Signal-Driven Orchestration}
The old rigid loop used fixed partition selection, fixed top-k, and fixed cadence for reflection and self-check. The current controller is signal-driven.
\section{Representative Signals}
\begin{itemize}
  \item repeated\_steps
  \item contradiction
  \item consecutive\_failures
  \item rapid\_health\_drop
  \item interruption\_recovery
  \item low\_health
  \item high\_pressure
  \item high\_noise
  \item abnormal\_growth
  \item low\_yield\_recall
\end{itemize}
\section{Controller Decisions}
The sidecar decides when to recall, why to recall, whether to store or block a write, when to reflect, when to run self-check, when to compact local operational state, and when to switch between normal, stability, and convergence.
\chapter{Traceability}
Every run should emit an append-only JSONL trace. The trace is expected to include at least:
\begin{itemize}
  \item event id
  \item request id
  \item trigger source
  \item replay flag
  \item replay attempt
  \item whether recall was requested
  \item recall reason
  \item backend used
  \item recall query
  \item returned memory count
  \item selected memory ids
  \item discard reasons
  \item health, pressure, noise, complexity
  \item current mode
  \item selected strategy
  \item reflection, self-check, and cleanup decisions
  \item whether memory write was allowed
  \item patch proposal and patch apply outcome
  \item runtime step
  \item ack id
  \item final outcome and failure reason
\end{itemize}

\chapter{Patch Safety}
Patch proposal may remain available, but automatic patch apply must remain off by default. Apply is only allowed behind explicit configuration. Proposal, apply, and failure must all be traced. Failure must not be silent.

\chapter{Migration Notes}
\begin{itemize}
  \item \texttt{knowledge/*.json} is no longer the sole source of truth for long-term memory.
  \item Local JSON snapshots remain, but only as operational state.

  \item Fixed-step orchestration has been replaced by signal-driven orchestration.
\end{itemize}

\chapter{Tradeoffs And Deferred Items}
\section{Main Tradeoffs}
\begin{itemize}
  \item thin bridge instead of a second memory engine in Python
  \item child-process model instead of a heavyweight daemon
  \item JSON fallback retained for compatibility but not treated as canonical when lancedb-pro is present
  \item append-only JSONL traces for readability and auditability
\end{itemize}
\section{Deliberately Deferred}
\begin{itemize}
  \item richer selected-memory and discard-reason explanations
  \item more advanced replay prioritization
  \item dedicated trace dashboard
  \item broader patch allowlist governance
\end{itemize}
